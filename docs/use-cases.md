# Юзкейси системи «Кулінарна книга»

Цей документ містить перелік юзкейсів для домену кулінарних рецептів відповідно до вимог лабораторної роботи.

---

## 1. Створення рецепта
* **Назва:** Створення нового рецепта
* **Актор:** Авторизований кухар
* **Передумови:** Користувач успішно пройшов автентифікацію та має активний токен.
* **Основний сценарій:**
  1. Користувач відправляє POST-запит із назвою страви, часом приготування та списком інгредієнтів.
  2. Система перевіряє інваріанти: назва не може бути порожньою, час приготування має бути більше 0.
  3. Система зберігає рецепт у базі даних, прив'язуючи його до автора.
  4. Система повертає статус 201 Created.
* **Альтернативні сценарії / помилки:**
  * Якщо час приготування менше або дорівнює 0, система повертає 400 Bad Request.
  * Якщо JWT-токен відсутній або недійсний, система повертає 401 Unauthorized.

## 2. Реєстрація користувача
* **Назва:** Реєстрація нового акаунта
* **Актор:** Гість
* **Передумови:** Користувач ще не зареєстрований у системі.
* **Основний сценарій:**
  1. Користувач вводить свій email та пароль.
  2. Система перевіряє валідність email.
  3. Система хешує пароль та створює новий запис у базі даних.
  4. Система повертає статус 201 Created.
* **Альтернативні сценарії / помилки:**
  * Якщо користувач із таким email вже існує, система повертає 409 Conflict.
  * Якщо email не відповідає стандарту, повертається 400 Bad Request.

## 3. Отримання списку рецептів
* **Назва:** Перегляд усіх рецептів
* **Актор:** Будь-який користувач
* **Передумови:** Відсутні.
* **Основний сценарій:**
  1. Користувач робить GET-запит на ендпоінт /recipes.
  2. Система витягує дані з бази даних.
  3. Система повертає статус 200 OK та масив об'єктів.
* **Альтернативні сценарії / помилки:**
  * Якщо запит іде на неіснуючий шлях, система повертає 404 Not Found.
